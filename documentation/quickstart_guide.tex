\section{Normal daily workflow}

This is a command-oriented reference for routine operation. Run the commands from a terminal opened in the Ubuntu graphical desktop or VNC session.

\begin{warningbox}[Before operating hardware]
Use Phoebus to change PSU or chiller settings. Do not use command-line PV writes unless you have been trained for the specific action. Stopping the software does not switch off hardware outputs or stop the chiller.
\end{warningbox}

\subsection*{1. Enter the repository}

\begin{lstlisting}[style=shell]
cd ~/SlowControl/app/bdx-slow-control
\end{lstlisting}

If an installed command is not found, activate the repository environment in that terminal:

\begin{lstlisting}[style=shell]
source .venv/bin/activate
\end{lstlisting}

\subsection*{2. Start the complete slow control}

\begin{lstlisting}[style=shell]
bdx_slow_control_start overview
\end{lstlisting}

\textbf{What it does:} starts or reuses the main IOC, checks whether the Raspberry temperature IOC is responding, reports the independent local Archiver Appliance status without changing it, and opens Phoebus on the overview page.

\textbf{What it does not do:} it never starts, stops, audits, repairs, or registers PVs in the Archiver. Use the dedicated \command{bdx_archiver_*} commands for those actions.

\textbf{Expected result:} the BDX Main IOC terminal remains open when a new IOC is started, and Phoebus displays the BDX overview. The command does not enable PSU outputs, start the chiller, or change setpoints.

\textbf{Implementation:} the installed command is declared in \file{pyproject.toml} and calls \file{src/bdx_slow_control/operator_startup.py}; the Phoebus terminal workflow sources \file{scripts/start_bdx_stack.sh} and launches through the repository Phoebus helpers.

\subsection*{3. Operate from Phoebus}

Use the navigation buttons in the overview page.

\begin{center}
\begin{tabularx}{\textwidth}{@{}l X@{}}
\toprule
Page & Use \\
\midrule
Overview & Overall status and navigation \\
PSU & Routine control of LV1 and LV2 \\
Chiller & Chiller status, start/stop, and setpoint \\
Environment & Raspberry temperature sensors \\
Trends & Live and archived plots \\
Global & Global status and confirmed all-off controls \\
\bottomrule
\end{tabularx}
\end{center}

Before changing a setting, check that the relevant communication indicator is healthy and verify the readback after the command.

\subsection*{4. Stop the local software stack}

\begin{lstlisting}[style=shell]
bdx_slow_control_kill
\end{lstlisting}

This gracefully stops the BDX Phoebus session and main IOC. It does not modify the independently managed Archiver Appliance, switch off PSU outputs, send an automatic all-off, stop the chiller, or change hardware settings.

\textbf{Implementation:} \file{src/bdx_slow_control/operator_commands.py} dispatches this command to \file{scripts/kill_slow_control_all.sh}.

\section{Startup commands}

\subsection{Open a specific Phoebus page at startup}

\begin{lstlisting}[style=shell]
bdx_slow_control_start psu
bdx_slow_control_start chiller
bdx_slow_control_start environment
bdx_slow_control_start trends
\end{lstlisting}

The same stack is started, but Phoebus opens directly on the selected page.

\subsection{Start the Raspberry temperature IOC}

\begin{lstlisting}[style=shell]
start-bdx-raspberry-ioc
\end{lstlisting}

Starts the Raspberry systemd service through SSH and waits for a temperature PV to respond. Run this when the main startup command reports that the Raspberry IOC is unavailable.

\subsection{Restart the Raspberry temperature IOC}

\begin{lstlisting}[style=shell]
start-bdx-raspberry-ioc --restart
\end{lstlisting}

Restarts the remote service and verifies that it is responding. The command is implemented by \file{src/bdx_slow_control/operator_commands.py}; it runs \command{systemctl start} or \command{systemctl restart} for \command{bdx-environment-ioc} on the Raspberry through SSH.

\subsection{Reopen Phoebus without starting another IOC}

\begin{lstlisting}[style=shell]
./scripts/launch_phoebus.sh overview
\end{lstlisting}

Use this when the IOC is already running and only the graphical interface must be reopened. Replace \code{overview} with \code{psu}, \code{chiller}, \code{environment}, \code{trends}, or \code{all_pvs}.

\section{Read-only checks}

\begin{lstlisting}[style=shell]
caproto-get BDX:GLOBAL:SYSTEM_STATE
caproto-get BDX:PSU:LV1:CH1:VOLTAGE_RBV
caproto-get BDX:ENV:TEMP:T00:VALUE
\end{lstlisting}

These commands read, respectively, the global state, the LV1 channel-1 measured voltage, and the first Raspberry temperature sensor. They do not change hardware settings.

\subsection{Check whether a device is reachable}

\begin{lstlisting}[style=shell]
nc -vz 172.22.50.20 9221    # LV1
nc -vz 172.22.50.21 9221    # LV2
nc -vz 172.22.50.60 54321   # chiller
\end{lstlisting}

A successful result means that the TCP control port is reachable. It does not prove that the device is operating correctly.

\section{Archiver commands}

The Archiver Appliance is managed independently from the main IOC and Phoebus:

\begin{longtable}{@{}p{0.25\textwidth}p{0.39\textwidth}p{0.30\textwidth}@{}}
\toprule
Command & Effect & Implementation \\
\midrule
\endhead
\command{bdx_archiver_start} & Start and validate the four Archiver components; audit and selectively repair the required catalog unless \code{--no-repair} is supplied & \file{src/bdx_slow_control/operator_startup.py}, using functions from \file{scripts/start_bdx_stack.sh} \\
\addlinespace
\command{bdx_archiver_audit} & Check the 18 required archived PVs without changing registrations & \file{src/bdx_slow_control/operator_startup.py}, dispatching \file{deploy/archiver-appliance/scripts/repair-archiver.sh} with \code{--audit-only} \\
\addlinespace
\command{bdx_archiver_repair} & Repair only missing, disconnected, or incorrectly configured required PVs & \file{src/bdx_slow_control/operator_startup.py}, dispatching \file{deploy/archiver-appliance/scripts/repair-archiver.sh} \\
\addlinespace
\command{bdx_archiver_kill} & Stop only the Archiver Appliance & \file{src/bdx_slow_control/operator_commands.py}, dispatching \file{scripts/kill_slow_control_archiver.sh} \\
\bottomrule
\end{longtable}

Live Channel Access control remains available if historical retrieval is unavailable. Archive history begins only after a PV is successfully registered and sampled; missing earlier data cannot be reconstructed.

\section{Shutdown commands}

\begin{longtable}{@{}p{0.29\textwidth}p{0.34\textwidth}p{0.31\textwidth}@{}}
\toprule
Command & Effect & Script used \\
\midrule
\endhead
\command{bdx_slow_control_kill_phoebus} & Stop only BDX slow-control Phoebus processes & \file{scripts/kill_slow_control_phoebus.sh} \\
\addlinespace
\command{bdx_archiver_kill} & Stop only the repository-managed local Archiver Appliance & \file{scripts/kill_slow_control_archiver.sh} \\
\addlinespace
\command{bdx_slow_control_kill_ioc} & Stop only the BDX main IOC & \file{scripts/kill_slow_control_ioc.sh} \\
\addlinespace
\command{bdx_slow_control_kill} & Stop Phoebus and the main IOC without modifying the Archiver Appliance & \file{scripts/kill_slow_control_all.sh} \\
\bottomrule
\end{longtable}

The installed entry points above are declared in \file{pyproject.toml}; \file{src/bdx_slow_control/operator_commands.py} validates the repository and forwards their arguments to the listed scripts.

\begin{warningbox}[Software shutdown is not hardware shutdown]
Put the equipment in the required state before stopping the IOC. None of these commands automatically switches off PSU outputs or stops the chiller.
\end{warningbox}

\section{Common problems}

\begin{longtable}{@{}p{0.33\textwidth}p{0.61\textwidth}@{}}
\toprule
Message or symptom & Command or action \\
\midrule
\endhead
\code{command not found} & Run \command{source .venv/bin/activate}, then retry. \\
\addlinespace
No graphical display is available & Run the startup command from the Ubuntu desktop or VNC session, not from a plain non-graphical SSH shell. \\
\addlinespace
Raspberry IOC is not responding & Run \command{start-bdx-raspberry-ioc}; use \command{start-bdx-raspberry-ioc --restart} when a restart is required. \\
\addlinespace
Phoebus was closed but the IOC is running & Run \command{./scripts/launch_phoebus.sh overview}. \\
\addlinespace
A PSU is unreachable & Run the corresponding \command{nc -vz} check for LV1 or LV2. \\
\addlinespace
The chiller is unreachable & Run \command{nc -vz 172.22.50.60 54321}. \\
\addlinespace
Archiver services are absent or degraded at slow-control startup & Start or validate them separately with \command{bdx_archiver_start}; the normal \command{bdx_slow_control_start} command intentionally performs only a read-only health report. \\
\addlinespace
Live values are present but history is missing & Run \command{bdx_archiver_audit}; repair with \command{bdx_archiver_repair} only after the Archiver components are healthy. Check the selected time range and host clock synchronization. \\
\addlinespace
A value is visible but no longer changes & Check the communication indicator in Phoebus and read the corresponding PV again with \command{caproto-get}. \\
\bottomrule
\end{longtable}
